# Triangle Route Diagnostics Design

**Context**

当前 Binance 三角链路的核心问题，不是已经确认“路由算错”，而是**证据不够**：

- `USD1-USDT` 主腿最近 live 日志里几乎没有真实成交
- `USD1-USDC` 释放腿也基本没有真实成交
- 共享路由账本还没有形成有效链路数据
- 当前状态快照里，`triangle_route_snapshot` 和 `triangle_exit_route_choice` 没有留下足够可解释信息

所以现在如果直接调阈值，只是在盲调。

**Options**

1. 直接下调阈值，强推更多间接路由  
   不选。缺点是没有证据，很容易把本来就薄的利润密度再压坏。

2. 直接做完整三腿自动执行  
   不选。改动面太大，当前还没到这个阶段。

3. 先补三角路由诊断与报告  
   选择。先把“为什么没有走间接路”“现在买入是否通过双出口门槛”“当前无路由是因为没库存还是没快照”这些问题记录清楚，再拿真实数据调参。

**Chosen Design**

本次只做诊断层，不改已有下单行为：

1. 在 bot 里生成三角诊断摘要
   - 快照是否可用
   - 快照是否过期
   - 当前库存是否为零
   - 当前若有库存，主路/备路是什么
   - 当前若按买一入场，双出口门槛是否放行
   - 若不放行，原因是什么

2. 把诊断摘要放进状态
   - 方便终端直接展示
   - 方便状态快照持久化
   - 方便报告工具后续读取

3. 把诊断变化写入 journal
   - 只在诊断结果发生变化时写，避免刷爆日志
   - 后续可统计：
     - 快照缺失次数
     - 平仓主路偏直出还是偏间接
     - 入场门槛被挡住的原因分布

4. 扩展现有路由报告
   - 即使当前没有 fills，也能看到：
     - 最近诊断状态
     - 入场门槛是否长期关闭
     - 当前链路到底卡在哪

**Out of Scope**

- 不做阈值下调
- 不做新三腿执行器
- 不改 release handoff 的成交逻辑
- 不改现有 entry / rebalance 的价格和数量

**Why This Order**

这一步的目标不是立刻多赚钱，而是把第 3 刀从“猜测三角链路没起作用”变成“能直接看到没起作用的具体原因”。  
只有这样，后面的阈值调整才不会误伤已经能跑的主路径。
